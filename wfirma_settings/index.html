    Первоначальная настройка¶

    Онлайн бухгалтерия wFirma первый год бесплатно 🪄¶

    Подписка для бухгалтерии ИП 2026: MALAKSIEGOWOSC-2026 (1 год) или PROX-KSIEGOWOSC (3 мес.) или K9D6ECEE29 (неизвестно) (подходит для ryczałt, skala podatkowa, liniowy).

    Регистрироваться только по этой ссылке ▶️ https://wfirma.pl/rejestracja-przez-kody ◀️

    Регистрация¶

    1. В поле Kod pakietu укажите код, чтобы получить бесплатный год.
    2. Укажите номер своего NIP при регистрации, чтобы сведения о вашем ИП были автоматически загружены из реестра.
    3. Прочитайте и примите правила сервиса (галочка Akceptuję regulamin serwisu).

    После регистрации переходим к настройкам.

    1. Нажимаем на иконку профиля.
    2. Нажимаем на Ustawienia (Настройки).

    Настройка данных фирмы¶

    Данные фирмы показываются на фактуре для заказчика. Скорее всего, wfirma сама предложит заполнить эти данные сразу после регистрации и не пустит дальше, пока вы их не внесёте.

    1. На панели настроек Moja firma выбираем Edytuj (Редактировать).

    2. В диалоговом окне DANE PODSTAWOWE - DANE IDENTYFIKACYJNE заполняем общие данные о себе (вкладка OGÓLNE), такие как:

      • полное название (Nazwa pełna)
      • сокращённое название (Nazwa skrócona)
      • NIP (только цифры, без префикса PL)
      • REGON
      • BDO (у простого программиста такого номера может не быть - это ок)
      • телефон
      • email

    3. Заполняем адрес (вкладка Adres).

      Укажите корректные данные 5 ваших адресов:

      Polski Русский
      adres prowadzenia działalności адрес ведения деятельности
      adres zameldowania адрес регистрации по месту жительства
      adres zamieszkania адрес проживания
      adres korespondencyjny почтовый адрес / адрес для корреспонденции
      adres na fakturze sprzedaży адрес на фактурах, к-е вы будете выставлять

      Если при регистрации JDG указывали "не имею постоянного места ведения деятельности", то можете заполнить адрес замешкания, например, адрес арендованной квартиры. В данном случае это нормально, т.к. данный адрес нам нужен для отображения на фактурах для заказчика.

    4. Переходим на вкладку банковских счетов (Rachunki bankowe).

    5. Нажмите кнопку Dodaj rachunek (Добавить счет).

    6. В диалоговом окне Dodawanie rachunku bankowego заполните данные о банковском счёте на обеих вкладках (Podstawowe informacje + Zaawansowane) и нажмите Zapisz (Сохранить).

      Можно добавить несколько банковских счетов. Затем при выставлении фактуры можете выбрать добавленный банковский счет, который будет показываться на фактуре для заказчика.

    Нажимаем Zapisz (Сохранить), чтобы сохранить данные фирмы. На email приходит письмо от wFirma с просьбой подтвердить изменение.

    Данные владельца фирмы¶

    Скорее всего wFirma сама предложит заполнить эти данные при попытке вычислить PIT и не пустит дальше, пока вы их не внесёте (но остаётся загадкой причина почему так).

    1. На панели настроек Dane podstawowe выбираем Rodzaj firmy i właściciele (вид бизнеса и владельцы).

    2. В диалоговом окне заполняем общие данные о себе как о владельце, такие как:

      • Rodzaj firmy: osoba fizyczna (индивидуальный предприниматель)
      • Imię, Nazwisko: имя и фамилия
      • obywatelstwo: гражданство (вроде не обязательно)
      • PESEL: только цифры, без пробелов
      • Rodzaj dokumentu, numer dokumentu: Вид документа, удостоверяющего личность (вроде не обязательно). Если указываете, то как иностранец выбирайте "paszport" (паспорт) и вводите номер паспорта (только латинские буквы и цифры без пробелов).
      • Data urodzenia: дата рождения в формате ГГГГ-ММ-ДД

    Нажимаем Zapisz, чтобы сохранить.

    Настройка налоговых данных¶

    Для этого нам понадобится блок Podatki.

    Podatki - Ogólne¶

    1. Выберите дату начала ведения JDG (data rozpoczęcia działalności gospodarczej), которую указали при регистрации.

    2. Выберите с какого месяца хотите начать считать налоги в wFirma (Miesiąc rozpoczęcia prowadzenia księgowości w systemie).

      Info

      Почти всегда совпадает с месяцем начала ведения деятельности.

      Для тех, кто переносит бухгалтерию из другой системы

      Учтите, что более старые документы (с датой раньше выбранного месяца) вы просто не сможете внести в wFirma. Тем из вас, кому не лень заносить исторические данные, рекомендуется указать здесь styczeń (январь) текущего года, чтобы полностью год был в одной системе.

    3. Номер индивидуального счёта - микросчёта, на который будут оплачиваться налоги (Indywidualne konto bankowe - mikrorachunek) - подтягивается автоматически. Рядом с полем может появиться кнопка "Подтвердить" (Potwierdź dane), нажмите её, чтобы подтвердить номер микросчёта.

      Info

      Проверить микросчёт можно с помощью официального налогового сервиса Generator mikrorachunku podatkowego по номеру NIP.

    Дополнительный шаг только для тех, кто ведёт KPiR и переносит бухгалтерию из другой системы не с начала года

    "Przenosisz księgowość w trakcie roku? Aby poprawnie wyliczyć zaliczkę uzupełnij sumy z KPiR z innego systemu. Stan na koniec..."

    Чтобы корректно рассчитать авансовый платёж, перенесите суммы из KPiR, подготовленной в другой системе. Состояние на конец ... (предыдущего месяца, см. подсказки от wFirma).

    Нажимаем Zapisz, чтобы сохранить.

    Podatki - ZUS¶

    Указывайте в настройках wFirma точно такие же данные, которые вы указывали при регистрации в ZUS. Расхождения в настройках wFirma и настройках ZUS - причина многих ошибок! Правильный источник данных: последний бланк ZUS ZZA или ZUS ZUA.

    1. Номер счёта, на который будут оплачиваться взносы ZUS (Indywidualne konto bankowe), подтягивается автоматически. Рядом с полем может появиться кнопка "Подтвердить" (Potwierdź dane), нажмите её, чтобы подтвердить номер микросчёта.

      Info

      Проверить можно с помощью официального сервиса e-Składka.

    2. Выберите схему взносов (Schemat składek społecznych).

      • Если вы только начинаете свои приключения с ИП, то скорее всего для вас это Ulga na start (льгота для начинающих; убедитесь, что имеете право на применение этой льготы!).
      • Если вы уже предприниматель со стажем, то дополнительно wFirma предложит выбрать "Добровольные больничные взносы: ДА/НЕТ" (Dobrowolne chorobowe).
      • Перечислим все доступные варианты:
      Polski Русский Пояснение
      Duży ZUS Полный ZUS Взносы на соцстрахование, здравоохранение и фонд труда. База для расчёта: 60% средней зарплаты. Код: 0510 или 0512.
      2 letni preferencyjny ZUS 2 года льготного ZUS Уменьшенные взносы на соцстрахование — 30% от минимальной зарплаты. Код: 0570 или 0572.
      Mały ZUS Plus Малый ZUS Плюс Для доходов до 120 000 злотых в прошлом году. Требует ручного ввода базы. Код: 0590 или 0592.
      Tylko zdrowotne Только медицинское страхование Для тех, кто работает по найму и параллельно ведёт бизнес — только медстраховка. Код: 0510 или 0512.
      Tylko zdrowotne - 6-miesięczna ulga na start Только медстраховка – 6 месяцев льготы на старт Начинающие предприниматели. Код регистрации в ZUS: 0540.
      Tylko zdrowotne - 2 letni preferencyjny ZUS Только медстраховка – 2 года льготы для новых фирм Новые фирмы с правом только на медстрахование. Код: 0570 или 0572.
      Tylko zdrowotne - mały ZUS plus Только медстраховка – Малый ZUS Плюс Упрощённая система с правом на только медстрахование. С 01.02.2020. Код: 0590 или 0592.
      Brak składek Без взносов Для застрахованных в KRUS или в другой стране ЕС — не нужно платить ZUS в Польше.
      Mały ZUS i Tylko zdrowotne - mały ZUS Малый ZUS и Только медстраховка – Малый ZUS Устаревшие схемы, действовали до 31.01.2020. Сейчас использовать нельзя.

      Расширенные настройки (Zaawansowane) установятся автоматически. Не трогайте их. В очень редких случаях, таких как ИП-пенсионер или ИП-инвалид, здесь нужно что-то менять. Подробнее об этом читайте в документации wFirma. 3. Выберите отдел NFZ (Oddział NFZ), который указывали при регистрации ZUS. Он изменяется при переезде в другое воеводство.

    Pro Tip!

    На подсказки (кнопки со знаком "?") можно нажимать правой кнопкой мыши и выбирать "Открыть в новой вкладке", чтобы открыть подробную статью, а не краткую выжимку, которая появляется при нажатии левой кнопкой.

    Нажимаем Zapisz, чтобы сохранить.

    Только для тех, кто переносит бухгалтерию из другой системы не с начала года

    Снова откройте окошко "Podatki - ZUS". Перейдите на вкладку "Składka zdrowotna", нажмите "Dodaj składnik" и заполните "Przychody narastająco" и "Suma składek społecznych zapłaconych od początku roku" в соответствии с подсказкой.

    Нажмите Zapisz, чтобы сохранить.

    Podatki - Podatek dochodowy¶

    1. Выберите Urząd skarbowy (налоговую инспекцию), которую указывали при регистрации JDG (или переезде). Если забыли, то проверить можно в своём кабинете предпринимателя с помощью стороннего инструмента CEIDG Viewer.

    2. Укажите вид бухучёта (Rodzaj ewidencji):

      Polski Русский
      Ewidencja Przychodów (ryczałt) Учёт Доходов (рычалт)
      Księga Przychodów i Rozchodów Книга Учёта Доходов и Расходов (PKPiR)
      Księgi rachunkowe﹡ Бухгалтерские книги﹡

      * Księgi rachunkowe

      Księgi rachunkowe - это полный учёт. Обязателен для тех, кто зарабатывает свыше 2.000.000 евро. Малые и средние предприниматели могут применять добровольно (но зачем?!).

      Плохие новости

      Если у вас никак не получается выбрать другие варианты, кроме Księgi rachunkowe, то есть варианты EP и KPiR заблокированы, то это из-за того, что вы использовали неправильный промокод для регистрации. Вам придётся удалить этот аккаунт wFirma и начать процесс настройки заново. Используйте те промокоды для JDG, которые есть в начале данной статьи.

    3. В поле DOMYŚLNA STAWKA RYCZAŁTU выберите процента налога, соответствующий вашему коду PKWiU. Если у вас несколько видов деятельности, то выберите преобладающую ставку, которая будет использоваться по умолчанию при выставлении фактур в системе (с возможностью изменить на другую).

    4. В поле Zaliczka выбираем ежемесячную (miesięczna) или ежеквартальную (kwartalna) оплату аванса PIT. Выбор совершается один раз и на весь год.

    5. Ставим галочку, если применяете Metoda kasowa w PIT. Пояснение:

      Polski Русский Объяснение
      Metoda kasowa w PIT Кассовый метод в PIT Учёт доходов и расходов по факту оплаты (на банковский счёт). Нужно заявить в налоговую о своём желании применять кассовый метод.
      Metoda memoriałowa w PIT Метод начислений в PIT Учёт доходов и расходов по моменту возникновения обязательства (начислению). Применяется по-умолчанию ко всем ИП.

    Нажимаем Zapisz, чтобы сохранить.

    Podatki - Podatek VAT¶

    Внимание! Настройки wFirma должны в точности соответствовать действительности. Рекомендуется выполнить простые проверки, чтобы избежать ошибок.

    1. Оставьте галочку płatnik VAT, если зарегистрированы на VAT в Польше.
      • Проверка:
        1. Зайдите на сайт podatki.gov.pl. Это biała lista.
        2. Впишите один из идентификаторов: Номер счёта (Numer konta), цифры NIP или REGON, используя свои данные.
        3. Нажмите синюю кнопку «Szukaj». Результат:
          • красный "Nie figuruje w rejestrze VAT" или зелёный "Status podatnika: Zwolniony" => Освобождён от VAT => ☐ НЕ отмечайте галочку.
          • зелёный "Status podatnika: Czynny" => Плательщик VAT => ☑ Отметьте галочку.
          • ошибка => проверьте, правильно ли ввели данные. Возможно, сервис временно не работает по техническим причинам.
    2. Выберите Urząd skarbowy (налоговую инспекцию), которую указывали при регистрации JDG (или переезде).

    3. Снимите галочку platnik VAT, если вы не зарегистрированы на VAT в Польше. А в поле "podstawa prawna zwolnienia z VAT" укажите причину освобождения.

      Polski Русский Пояснение
      Zwolnienie ze względu na rodzaj prowadzonej działalności Освобождение в связи с видом осуществляемой деятельности Некоторые виды деятельности (например, образовательные услуги) освобождены от НДС по закону Art. 43. - Zwolnienia przedmiotowe.
      Zwolnienie ze względu na nieprzekroczenie 240 000 PLN obrotu Освобождение в связи с непревышением оборота в 240 000 злотых Если годовой доход не превышает 240 000 PLN, можно не становиться плательщиком НДС по закону Art. 113. - Zwolnienia podmiotowe. ⭐ Популярная причина.
      Zwolnienie na mocy rozporządzenia MF Освобождение на основании постановления Минфина Особый случай.
      Inna podstawa prawna Другое правовое основание Очень особый случай.

    4. Поставьте галочку przedsiębiorca zarejestrowany w UE (VAT-UE), если зарегистрированы на VAT в ЕС (требуется, например, если оказываете услуги фирмам из ЕС).

      • Проверка:
        1. Зайдите на сайт поисковой системы VIES EU.
        2. Заполните обязательные поля «Member State» (Państwo Członkowskie) и «VAT Number» (Numer VAT, т.е. NIP), введя свои данные.
        3. Нажмите синюю кнопку «Verify» (Weryfikuj). Результат:
          • красный "Nieważny numer VAT dla transakcji transgranicznych w obrębie UE" => Не зарегистрирован на VAT UE => ☐ НЕ отмечайте галочку.
          • зелёный "Tak, numer VAT aktywny" => Плательщик VAT UE => ☑ Отметьте галочку.
          • ошибка => проверьте, правильно ли ввели данные. Возможно, сервис временно не работает по техническим причинам.

    Podatki - Funkcje księgowe¶

    1. В меню для настройки бухгалтерских функций включаем функцию расчёта курсовой разницы (wyliczanie różnic kursowych).

      Пригодится, если получаете оплату в иностранной валюте и дата в фактуре (дата выставления или дата продажи - что раньше) отличается от даты зачисления оплаты.

    2. А функция "Sprawdzanie poprawności dat księgowania" нужна для того, чтобы wFirma проверяла корректность дат бухгалтерских записей.

    В большинстве случаев рекомендуется оставить галочку. Снятие галочки позволяет ввести в систему документ, выставленный до даты начала использования системы wFirma, для которого дата бухучёта попадает в период, когда бухгалтерский учёт уже ведётся в системе.

    Настройка услуг¶

    Это каталог. Добавленные в него услуги можно будет выбрать при выставлении фактуры для автоматического заполнения полей.

    1. Нажимаем Przychody или Magazyn (зависит от пакета wFirma).
    2. Нажимаем Produkty.
    3. Нажимаем Dodaj produkt.

    4. Вводим название услуги (Nazwa). Важно выбрать то, что соответствует вашему роду деятельности и коду PKWiU. Обычно для программистов это могут быть:

      • Usługi komputerowe
      • Tworzenie oprogramowania
      • Usługi oprogramowania

      Для предпринимателей, ведущих внешнеэкономическую деятельность, рекомендуем указывать двойное название: польский + иностранный язык. Иностранный язык - для вашего контрагента. Польский язык - для местных ужондов.

      • Tworzenie oprogramowania | Software development

    5. Выбираем тип (Typ) - Usługa.

    6. Выбираем единицу измерения по-умолчанию (Jednostka podst.) - usł.
    7. (опционально) Вводим соответствующий код PKWiU/CN. Значение кода на скриншоте:
      • 62.01.1 = Usługi związane z projektowaniem i rozwojem technologii informatycznych = Услуги, связанные с проектированием и разработкой информационных технологий.
    8. (если ryczałt) Выбираем соответствующую ставку налога. Для PKWiU 62.01.1 это 12%.

    Также, сразу можно указать цену нетто / брутто (плательщики VAT) или просто цену (Cena; в случае плательщиков, освобождённых от VAT). Но тут есть проблема, что на данном этапе нет возможности указать валюту. И если укажете цену, а потом в фактуре поменяете валюту, то цена в фактуре может автоматически конвертироваться. Поэтому цену можно пропустить и указывать при выставлении фактуры.

    Интеграция с KSeF¶

    KSeF - это национальная система электронных счетов-фактур (Krajowy System e-Faktur). Это государственный реестр, в который отправляются фактуры, выставляемые в Польше. Из него же получаются фактуры от польских фирм.

    1. Нажимаем Przychody.
    2. Нажимаем KSeF.
    3. Нажимаем Konfiguracja integracji.

    Дальше действуем по обстоятельствам (следуйте оф. инструкции на экране). В целом, нужно будет использовать уже имеющийся сертификат или создать пару новых сертификатов, скачать их и загрузить в wFirma. Процесс несложный, но требует внимательности.

    Настройка контрагентов¶

    Добавляем контрагента, кому будем выставлять фактуры. Добавленного контрагента можно будет выбрать при выставлении фактуры для ее автоматического заполнения. Данные настройки индивидуальны. Примерные шаги такие:

    1. Нажимаем CRM.
    2. Нажимаем Kontrahenci.
    3. Нажимаем Dodaj kontrahenta.
    4. Если контрагент из Польши, то в окошке Dodawanie nowego kontrahenta вводим его NIP и нажимаем синюю кнопку Pobierz. Данные подтянутся автоматически.
    5. Иначе можно заполнить данные вручную (кнопка WYPEŁNIJ RĘCZNIE).
    6. Например, если контрагент из другой страны ЕС (не Польша), то в окошке Dodawanie nowego kontrahenta можно выбрать «VAT UE» в качестве идентификатора, в поле Identyfikator добавить номер VAT-UE включая две буквы, с особой внимательностью заполнить адрес (если в адресе отсутствует почтовый индекс — ввести пробел в поле Kod), а также выбрать страну из выпадающего списка «Kraj». Вот список стран, которые относятся к группе «VAT UE».
    7. При добавлении нового иностранного контрагента из-за пределов ЕС, после ввода названия компании, необходимо выбрать тип идентификатора «Inne», заполнить адрес (если в адресе отсутствует почтовый индекс — добавить пробел), а также выбрать страну из выпадающего списка «Kraj». США или Великобритания, например, относятся к этой группе «Inne».

    На вкладке Faktury нет обязательных для заполнения полей. Но для вашего же удобства рекомендуется сразу указать:

    1. Rachunek - ваш счёт, который будет виднеться на фактурах в данных продавца для этого конкретного контрагента. Полезная настройка для тех бизнесов, которые имеют несколько разных счетов и используют разные счета для разных групп контрагентов.
    2. Język - язык фактуры. В данном случае выбран польский-английский (pol-ang).
    3. Metoda płatności - метод оплаты. На скриншоте выбран przelew - это банковский перевод.
    4. Szablon stałej informacji - шаблон постоянной информации — здесь можно добавить описание, которое будет отображаться на каждой фактуре для данного контрагента. Оставляем пустым.
    5. Rabat - скидка. Пропускаем.
    6. Termin płatności - срок оплаты. Присутствует возможность установить индивидуальный срок оплаты для контрагента, который будет отсчитываться в днях от даты выставления фактуры.
    7. Опция przypominaj o niezapłaconych fakturach - напоминания о неоплаченных фактурах. Если здесь не стоит галочка, то wFirma не будет отправлять напоминания данному контрагенту, даже если в глобальных настройках указано иное.

    Нажимаем Zapisz, чтобы сохранить данные, или Anuluj (отмена), чтобы закрыть окошко без сохранения.

    Настройка фактур¶

    Нумерация фактур¶

    В блоке Faktury => Serie numeracji можно добавить кастомную серию нумерации для фактур.

    1. Нажмите на кнопку Dodaj serię (Добавить серию).
    2. В поле Nazwa введите произвольное название.
    3. В поле Typ dokumentu выберите "Faktura VAT" - плательщики VAT; или "Faktura (bez VAT)" - все остальные.
    4. В поле Wzorzec введите шаблон номера. Например, "[numer]/[rok]", если вы хотите, чтобы ваши фактуры нумеровались в формате "1/2026" вместо дефолтного "FV 1/2026". Доступные плейсхолдеры вы найдёте, нажав на кнопку со знаком "?" (подсказка).
    5. В поле Numer początkowy введите начальный номер для этой серии. Обычно "1", но при переход с другой системы выставления инвойсов вы можете захотеть начать с другого номера, чтобы сохранить нумерацию непрерывной.
    6. "Resetowanie numeru: co rok" - сбрасывать нумерацию каждый год (разумное значение по-умолчанию).

    Для сохранения изменений нажмите Zapisz два раза.

    Шаблон фактуры¶

    В блоке Faktury => Szablony на вкладке Wybór szablonów можно выбрать из нескольких преднастроенных шаблонов фактур.

    • кнопка "Podgląd" (просмотр) позволяет увидеть, как будет выглядеть фактура.
    • синяя кнопка "Wybierz" (выбрать) позволяет выбрать шаблон, который вы хотите использовать по умолчанию. Вы всегда сможете изменить шаблон для конкретной фактуры на этапе её выставления.

    Шаблон письма¶

    В блоке Faktury => Powiadomienia на вкладке Email можно изменить шаблон сопроводительного письма, которое будет использовано при отправке фактуры клиенту.

    1. Найдите блок "Szablon wiadomości przy wysyłaniu faktury"
    2. Нажмите кнопку "Dodaj szablon wiadomości"
    3. В поле Nazwa введите произвольное название.

    4. Temat (Subject):

      Invoice no. [faktura] issued by [wystawca]

    5. Body (Treść):

      Please find attached invoice [faktura]. The value of the invoice is [razem], left to pay: [pozostało], the payment date is [termin]. Please print attached document.

    6. Отметьте галочку "domyślny szablon" (шаблон по умолчанию), если хотите, чтобы этот шаблон всегда использовался при автоматической отправке фактур.

    Для сохранения изменений нажмите Zapisz два раза.

    Подпись на фактуре¶

    В блоке Faktury => Logo firmy можно добавить изображение своей подписи или печати (Pieczątka / podpis) и логотип фирмы (Logo firmy), которые будут показываться на фактуре.

    Верификация фирмы¶

    Для отправки деклараций ZUS DRA из wfirma необходимо верифицировать профиль. Это можно сделать в блоке Dane podstawowe => Weryfikacja firmy. В целях подтверждения личности требуется отправить один перевод со своего бизнес-счёта.

    Info

    Стоит отметить, что верификация не является обязательной. Wfirma предоставляет один год бесплатного использования при регистрации. Если вы не верифицируете профиль, то через год сможете зарегистрироваться заново и получить ещё один бесплатный год (разумеется, если такая акция всё ещё будет доступна). Стоит ли это усилий — решать вам. Однако в таком случае вам придётся самостоятельно скачивать файл ZUS DRA и загружать его на портал ZUS.

    Перенос бухгалтерии¶

    Особенности для тех, кто переносит бухгалтерию, читайте в гайде Миграция на wFirma.

    Резервное копирование¶

    В окошке Bezpieczeństwo => Kopie zapasowe нажмите на кнопку "Stwórz kopię" для создания резервной копии настроек аккаунта. Резервные копии создаются автоматически, но эта кнопка позволяет создать резервную копию в любой момент. Операция восстановления данных является рискованной и должна использоваться только в крайнем случае.
