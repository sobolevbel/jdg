# Миграция на inFakt

Данный документ описывает перенос бухгалтерии в программу [inFakt.pl][1] из другого
сервиса (wFirma, iFirma, бухгалтер и т.д.) в середине года. После первоначальной
настройки нужно корректно перенести историю текущего года — иначе inFakt неправильно
посчитает налоги, ставку składki zdrowotnej и лимиты.

## Для кого

Подойдёт для типичного фрилансера JDG-шника на ryczałt с одной-двумя фактурами в месяц
за услуги (программирование). Для плательщиков VAT есть отдельные примечания по переносу
nadwyżki VAT.

## Подготовка

Первоначальная настройка inFakt описана в гайде [«Настройка infakt.pl»][2]. При миграции
обратите особое внимание на шаги мастера настройки, где inFakt спрашивает:

- **месяц, с которого ведёте учёт в inFakt** — укажите месяц миграции, а не начала деятельности;
- **сумму дохода с начала года** — сумма из вашей ewidencji przychodów в старой системе
  на конец последнего месяца, посчитанного там;
- **доход за предыдущий год** — понадобится для авторизации отправки деклараций
  (см. [ошибку «Weryfikacja negatywna»][3]) и вносится в
  Ustawienia -> Księgowość -> Podatek dochodowy -> Przychód w roku N:

    ![Ustawienia — Przychód w roku][9]

## Теория

Чтобы первый месяц в inFakt посчитался правильно, за него (и за прошедшую часть года)
нужно перенести:

1. **ДОХОД прошлого года и прошедших месяцев текущего года** — влияет на ставку składki
   zdrowotnej (пороги 60 000 / 300 000 zł) и лимит освобождения от VAT (240 000 zł).
2. **ДОХОД: фактуры отчётного месяца** — с продолжением нумерации из старой системы.
3. **ДОХОД: курсовые разницы**, возникшие в отчётном месяце по прошлогодним фактурам.
4. **ВЗНОСЫ ZUS** — настроить так, чтобы рассчитанные inFakt суммы совпали с фактически
   оплаченными (взносы в inFakt вычисляются автоматически, руками их не внести).
5. **Доплату składki zdrowotnej за прошлый год** (roczne rozliczenie), если она была
   оплачена в текущем году.

## Практика

### Перенос дохода прошедших месяцев

Доход, заработанный до перехода в inFakt, вносится как **dowód wewnętrzny** (внутренний
документ) — по одному на каждый прошедший месяц либо одним документом на сумму
с начала года.

Благодаря этому inFakt будет правильно определять ставку składki zdrowotnej и следить
за лимитами, а суммы в ewidencji przychodów совпадут со старой системой.

### Внесение фактур текущего месяца

Выставляйте фактуры по инструкции [«Выставление фактур»][4], но проверьте **нумерацию**:
она должна продолжать существующую серию из старой системы (например, если последняя
фактура была `12/2026`, следующая должна быть `13/2026`). Номер можно поменять вручную
при создании фактуры, а формат нумерации настраивается в **Ustawienia -> Faktury ->
Numeracja**:

![Ustawienia — Numeracja][7]

Если фактура уже оплачена — сразу отметьте её оплаченной и внесите дату оплаты, чтобы
inFakt корректно посчитал курсовые разницы (для оплат в валюте).

### Взносы ZUS

В inFakt оплаченные взносы ZUS нельзя просто внести руками — приложение **вычисляет их
само** на основании вашего дохода и настроек (система налогообложения, режим взносов
ulga na start / preferencyjne / duży ZUS, добровольное chorobowe, даты переходов между
режимами — см. [настройку данных ZUS][10]). Задача при миграции — выставить настройки
так, чтобы рассчитанные inFakt суммы **совпали с реально оплаченными** взносами из
старой системы.

Сверьте помесячно расчёт inFakt с фактическими оплатами (по банковской выписке).
Обратите внимание на два момента, где расчёты чаще всего расходятся:

1. **Декабрьские składki społeczne, оплаченные в январе.** По кассовому принципу они
   относятся к январю нового года и уменьшают налоговую базу января — проверьте, что
   inFakt учёл их именно так.
2. **Доплата składki zdrowotnej за прошлый год** (по итогам rocznego rozliczenia,
   оплачивается обычно в мае) — должна попасть в расчёты месяца фактической оплаты.

Экран взносов: **Księgowość -> Składki ZUS** — здесь видны периоды, статусы оплат
и суммы (в том числе строка «w tym z rozliczenia» у месяца с доплатой за roczne
rozliczenie zdrowotnej).

![Księgowość — Składki ZUS][8]

### Nadwyżka VAT (для плательщиков VAT)

Если вы czynny плательщик VAT и в последней декларации JPK_V7M из старой системы была
kwota nadwyżki VAT do przeniesienia (излишек VAT к переносу), её нужно указать в inFakt,
чтобы она попала в первую декларацию, отправленную из новой системы.

### Сверка

После внесения всех документов сравните с данными старой системы:

- сумму в **ewidencji przychodów** нарастающим итогом;
- рассчитанный **ryczałt/PIT** за первый месяц в inFakt;
- ставку и сумму **składki zdrowotnej** в декларации ZUS DRA.

Если всё сходится — миграция прошла успешно.

!!! warning "Не отправляйте декларации повторно"
    Декларации ZUS DRA и JPK, уже отправленные из старой системы, повторно отправлять
    не нужно. В inFakt их можно пометить как учтённые/оплаченные без отправки.
